---
title: "Troubleshoot Fluxcord bridges"
description: "Solutions for common Fluxcord issues: missing messages, failed bridges, attachment errors, voice not working, and bot not responding."
---

Most Fluxcord problems fall into a handful of categories: the bot not reacting to commands, messages not crossing the bridge, or voice sessions failing to start. Work through the relevant accordion below to find the cause and fix.

<AccordionGroup>
  <Accordion title="Bot not responding to commands">
    Check the following:

    - **Prefix** — the default prefix is `fc!`. If your config uses a different `BotPrefix`, use that instead.
    - **Bot online** — confirm the bot appears online in both Discord and Fluxer. If it shows offline, check that the process is running and that `DiscordBotToken` and `FluxerBotToken` in `config.js` are correct.
    - **Permissions** — the bot needs **Send Messages** and **Manage Webhooks** in the channel. Check server and channel permission overrides.
    - **Elevated commands** — commands like `fc!setup`, `fc!bridge`, `fc!verify`, and `fc!unbridge` require **Manage Server** permission (or your account ID in `AdminAccountIds`).
  </Accordion>

  <Accordion title="Bridge setup code expired">
    Pairing codes generated by `fc!setup` expire after **5 minutes**. If you see:

    > Code can't be found or is expired already.

    Run `fc!setup` again on the originating side to get a fresh code, then run `fc!setup <newcode>` on the other side before it expires.
  </Accordion>

  <Accordion title="Messages not flowing across the bridge">
    If the bridge is set up but messages aren't appearing:

    - **Check bridge direction** — if you used `discord2fluxer`, messages only flow from Discord to Fluxer. Messages sent on the Fluxer side will not appear on Discord. See [Bridge types and direction](/guides/bridge-types) for a full breakdown.
    - **Check the channel is bridged** — run `fc!setup` in the channel. If it says the channel is already bridged, the bridge exists. If it doesn't say that, the channel was never successfully bridged.
    - **Webhook deleted** — if a Fluxcord-managed webhook was manually deleted in Discord or Fluxer, messages from that direction will stop flowing. Run `fc!unbridge` and set up the bridge again to recreate the webhooks.
  </Accordion>

  <Accordion title="Attachments missing or corrupted">
    Attachment forwarding can fail when the CDN on either platform is unstable or returns a corrupted file. This is a known issue with no workaround on Fluxcord's side:

    - **Fluxer CDN failure** — the attachment appears corrupted or missing on the Fluxer side.
    - **Discord CDN failure** — the attachment appears missing on the Discord side.

    If attachments are consistently failing, check whether the originating platform's CDN is having issues. Fluxcord will link oversized attachments as text rather than uploading them (over ~25 MB for Discord→Fluxer, over ~10 MB for Fluxer→Discord).
  </Accordion>

  <Accordion title="Error reaction appearing on messages">
    When Fluxcord encounters an error processing a message, it may react to the message with an error indicator. To see the full error details, enable error logging in your guild:

    ```
    fc!guild seterrorlogging <channel>
    ```

    The bot will post error details to the specified channel, which can help identify whether the issue is a CDN failure, a missing permission, or something else.
  </Accordion>

  <Accordion title="Bridge rejected due to NSFW mismatch">
    If you see:

    > Both channels needs to be set as NSFW to bridge them.

    Both the Discord channel and the Fluxer channel must have the same NSFW status. Either mark both channels as NSFW, or make sure neither is. Then run `fc!setup` again.

    Note: due to a Fluxer platform limitation, NSFW Fluxer channels cannot be bridged to non-NSFW Discord channels regardless of configuration.
  </Accordion>

  <Accordion title="Voice bridging not working">
    Voice bridging requires additional setup beyond the core bot. Check each of the following:

    - **Voice runner running** — at least one voice runner process must be connected to the WebSocket server. If no runner is connected when a user joins, the bot logs `No runner available` and does not join the VC.
    - **RunnerSecret matches** — the `RunnerSecret` in `config.js` must match the `x-runner-secret` header sent by the voice runner. A mismatch causes the runner connection to be rejected with `4001 Unauthorized`.
    - **VoiceChannelMaps configured** — the channel the user joined must appear in `VoiceChannelMaps` with the correct `discordGuildId`, `discordChannelId`, `fluxerGuildId`, and `fluxerChannelId`.
    - **voice/node_modules installed** — run `cd voice && pnpm install` (or `npm install`) to install voice runner dependencies before starting runners.
    - **Enable VOICE logging** — add `"VOICE"` to `LoggingCategories` in `config.js` to see detailed voice session logs.
  </Accordion>

  <Accordion title="Edits from Discord not appearing on Fluxer">
    This is a known limitation. Due to Fluxer platform constraints, message edits originating on Discord are **not** forwarded to Fluxer. Edits made on Fluxer do bridge to Discord.

    There is no workaround. The original message on the Fluxer side will remain as it was when first bridged.
  </Accordion>

  <Accordion title="Bot restarts or disconnects after 10 seconds">
    If the Fluxer bot disconnects shortly after starting, the most likely cause is an invalid or missing `FluxerBotToken` in `config.js`. The Fluxer client will time out trying to connect to the gateway and the process may exit or restart.

    Double-check that `FluxerBotToken` is set to a valid Fluxer bot token. If you're using a self-hosted Fluxer instance, also verify that `FluxerAPIBaseURL` points to the correct API endpoint.
  </Accordion>
</AccordionGroup>

<Tip>
  Add `"DEBUG"` to the `LoggingCategories` array in `config.js` to enable verbose logging. This can surface errors that aren't visible at the default log level, including database queries and handler internals.
</Tip>
